Como usar el token

Usa el token en el header Authorization como Bearer. Ejemplo basico (usa NewserLite, 1 credito):

curl -X POST https://TU-DOMINIO/api/v1/chat \
  -H "Authorization: Bearer verboai-XXXXXXXXXXXXXXXXXXXXXXXX" \
  -H "Content-Type: application/json" \
  -d '{"mensaje":"Hola, quien eres?"}'

Para usar NewserAvanced (modelo mas potente, consume 5 creditos y rate limit 5/min), agrega "modelo" al body:

curl -X POST https://TU-DOMINIO/api/v1/chat \
  -H "Authorization: Bearer verboai-XXXXXXXXXXXXXXXXXXXXXXXX" \
  -H "Content-Type: application/json" \
  -d '{"mensaje":"Explicame el libro de Job","modelo":"NewserAvanced"}'

Respuesta (JSON):

{
  "ok": true,
  "respuesta": "Hola, soy NewserLite...",
  "modelo": "NewserLite",
  "modeloUsado": "NewserLite",
  "costoCreditos": 1,
  "creditosRestantes": 999,
  "rateLimitMax": 20,
  "rateLimitVentanaMs": 60000
}

Tambien podes consultar el estado del token y la lista de modelos disponibles con GET /api/v1/info (no gasta creditos):

curl -H "Authorization: Bearer verboai-XXXX" https://TU-DOMINIO/api/v1/info
  • Cada token arranca con 1000 creditos. NewserLite consume 1 por pedido, NewserAvanced consume 5.
  • Rate limit (API): 20/min para NewserLite, 5/min para NewserAvanced (ventana deslizante por token).
  • Rate limit (web, no consume creditos): 30/min para NewserLite, 8/min para NewserAvanced.
  • NewserAvanced consume mas creditos y tiene rate limit mas estricto (mas potente, mas lento).
  • NewserAvanced tiene herramientas exclusivas:
  • Generar imagen: escribi Generame [descripcion] como mensaje (en web o API). Consume +1 credito adicional. Se abre en el visor en terminal.
  • [[WEB::consulta]] busca en Google Custom Search — consume +1 credito adicional.
  • [[CLIMA::lugar]] consulta open-meteo — gratis, no consume creditos extra.
  • Si mandas un "modelo" invalido o vacio, cae automaticamente a NewserLite (no rompe).
  • Cuando se acaban los creditos, el token devuelve 402. Si queres mas, avisale al admin.
  • Borrar un token lo revoca de inmediato: cualquier peticion posterior con ese token devuelve 401.

Usar desde la terminal: descarga el archivo cli/verboai.py y ejecutá:

# 1. Guardar el token (una sola vez)
python verboai.py login verboai-TU_TOKEN

# 2. Iniciar chat interactivo
python verboai.py run

El CLI levanta un servidor proxy local en http://127.0.0.1:7788 (alta seguridad, el token nunca se expone).

Codigo completo para conectar desde tu app (sin CLI):

Copiá el que corresponda a tu lenguaje. Reemplaza verboai-XXXX por tu token y TU-DOMINIO por la URL de la app.

JavaScript (Node.js o navegador) 
// JavaScript - funciona en Node.js 18+ y en navegadores
const TOKEN = "verboai-XXXX";
const URL = "https://TU-DOMINIO/api/v1/chat";

async function preguntar(mensaje, modelo = "NewserLite") {
  const resp = await fetch(URL, {
    method: "POST",
    headers: {
      "Authorization": `Bearer ${TOKEN}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({ mensaje, modelo }),
  });
  const data = await resp.json();
  if (!data.ok) throw new Error(data.error);
  return data;
}

// Uso:
preguntar("Hola, quien eres?", "NewserLite")
  .then(r => console.log(r.respuesta, r.creditosRestantes))
  .catch(e => console.error(e));

// Generar imagen:
preguntar("Generame un leon descansando", "NewserAvanced")
  .then(r => {
    console.log(r.respuesta);
    if (r.imagen) console.log("Imagen:", "https://TU-DOMINIO" + r.imagen.url);
  });
Python (sin instalar nada) 
# Python 3.7+ - sin dependencias externas (solo stdlib)
import json
import urllib.request

TOKEN = "verboai-XXXX"
URL = "https://TU-DOMINIO/api/v1/chat"

def preguntar(mensaje, modelo="NewserLite"):
    body = json.dumps({"mensaje": mensaje, "modelo": modelo}).encode()
    req = urllib.request.Request(URL, data=body, method="POST")
    req.add_header("Authorization", f"Bearer {TOKEN}")
    req.add_header("Content-Type", "application/json")
    with urllib.request.urlopen(req, timeout=120) as resp:
        data = json.loads(resp.read().decode())
    if not data.get("ok"):
        raise Exception(data.get("error"))
    return data

# Uso:
r = preguntar("Hola, quien eres?", "NewserLite")
print(r["respuesta"], "creditos:", r["creditosRestantes"])

# Generar imagen:
r = preguntar("Generame un leon descansando", "NewserAvanced")
print(r["respuesta"])
if r.get("imagen"):
    print("Imagen:", "https://TU-DOMINIO" + r["imagen"]["url"])
PHP 
<?php
// PHP 5.6+ con cURL
$TOKEN = "verboai-XXXX";
$URL = "https://TU-DOMINIO/api/v1/chat";

function preguntar($mensaje, $modelo = "NewserLite") {
    global $TOKEN, $URL;
    $body = json_encode(["mensaje" => $mensaje, "modelo" => $modelo]);
    $ch = curl_init($URL);
    curl_setopt($ch, CURLOPT_POST, true);
    curl_setopt($ch, CURLOPT_POSTFIELDS, $body);
    curl_setopt($ch, CURLOPT_HTTPHEADER, [
        "Authorization: Bearer $TOKEN",
        "Content-Type: application/json",
    ]);
    curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
    curl_setopt($ch, CURLOPT_TIMEOUT, 120);
    $resp = curl_exec($ch);
    curl_close($ch);
    return json_decode($resp, true);
}

// Uso:
$r = preguntar("Hola, quien eres?", "NewserLite");
echo $r["respuesta"] . "\n";
echo "Creditos: " . $r["creditosRestantes"] . "\n";

// Generar imagen:
$r = preguntar("Generame un leon descansando", "NewserAvanced");
if (isset($r["imagen"])) {
    echo "Imagen: https://TU-DOMINIO" . $r["imagen"]["url"] . "\n";
}
?>
